Privy Cross-App Connect — Missing Origin Validation PoC

CRITICAL CWE-346

This page demonstrates that a malicious website can inject fake cross-app connection responses into any application using @privy-io/cross-app-connect.

Vulnerability Summary

The SDK opens a popup window and registers a window.addEventListener('message', ...) handler to receive the connection response. The handler checks event.data.type === 'PRIVY_CROSS_APP_CONNECT_RESPONSE' but does not validate event.origin. Any window, frame, or tab can therefore send a spoofed response.

Attack Flow

  1. A legitimate app uses @privy-io/cross-app-connect to initiate a wallet connection.
  2. The SDK opens a popup and listens for postMessage responses.
  3. The message handler does NOT check event.origin.
  4. An attacker page (any origin) sends a crafted PRIVY_CROSS_APP_CONNECT_RESPONSE.
  5. The attacker supplies their own ECDH public key.
  6. The SDK derives a shared secret using the attacker's key — the attacker can also derive it.
  7. All subsequent encrypted cross-app messages can be read and forged by the attacker.
